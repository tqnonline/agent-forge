# agent-forge v1.0.2

**Released: 2026-05-26**

Bug-fix patch for v1.0.1. Single user-visible fix: `agent-forge --version`
now works.

## Fixed
- **`agent-forge --version` crashed** with `RuntimeError: 'agent_forge' is
  not installed`. Click's default `version_option` looks up the package via
  `importlib.metadata.version(prog_name)`, using the CLI command name
  (`agent-forge`) — but the PyPI distribution is `tqn-agent-forge`, so the
  lookup failed. Now explicit: `version_option(package_name="tqn-agent-forge")`.

## Install

```bash
pipx install tqn-agent-forge==1.0.2   # fresh install
pipx upgrade tqn-agent-forge          # upgrade from 1.0.1
```

All other commands (`agent-forge install`, `update`, `list`, `detect`, etc.)
worked correctly in 1.0.1 — only `--version` was broken. No schema or API
changes.
